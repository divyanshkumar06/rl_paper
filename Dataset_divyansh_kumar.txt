3. PARTIAL VS FULL DATASET
--------------------------
- Training is performed on the FULL generated dataset (typically 2000 multi-turn dialogues for robust convergence).
- Each dialogue consists of 5-10 transition pairs (state, action, reward, next_state).

4. TRAINING & SIMULATION PARAMETERS
-----------------------------------
- State Representation (409D):
    - 25D Behavioral Dimensions: Frustration, Confidence, Correctness, etc. (Extracted via LLM classification).
    - 384D Semantic Dimensions: Domain context (Mathematics, Physics, Chemistry, etc.) extracted via Sentence-BERT.
- Action Classes (Discrete Action Space):
    0. Instruct (Partial Hint)
    1. Encourage (Motivational Support)
    2. Verify (Sub-step logic check)
    3. Guide (Socratic Guiding Question)
- Reward Shaping:
    - Sparse Reward: +1.0 for final success.
    - Dense Reward: +0.2 for mathematical effort (writing formula), -0.1 for frustration.
- Simulation Personas:
    - Shy: Highly hesitant, low confidence.
    - Overconfident: High confidence, avoids sub-step checks.
    - Distracted: Digresses to random topics.
